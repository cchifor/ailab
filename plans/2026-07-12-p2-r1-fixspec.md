# P2 R-1 fix contract (addresses the codex R-1 Phase B blockers)

The single shared contract all three repos fix against. Re-review after.

## 1. Env contract — the renderer emits EXACTLY these (matching agentforge `Settings`, env_prefix `AF_`)
`AF_EXECUTOR=sandbox`, `AF_SANDBOX_NAMESPACE=agentforge-sandbox`, `AF_SANDBOX_IMAGE=<digest>`,
`AF_SANDBOX_SERVICE_ACCOUNT=agentforge-sandbox`, `AF_SANDBOX_WORKSPACE_PVC=af-sbx-ws-<ws>`,
`AF_SANDBOX_WORKSPACE=<ws>`, `AF_SANDBOX_ACTIVE_DEADLINE_S`, `AF_SANDBOX_TTL_AFTER_FINISHED_S`,
`AF_SANDBOX_TERMINATION_GRACE_S`, `AF_SANDBOX_HOME_SIZE_LIMIT`,
`AF_SANDBOX_STAGING_ROOT=/workspace/agentforge/sandbox`, `AF_LEASE_TTL_S`,
`AF_LEASE_HMAC_KEY=<from the SAME Secret the reaper mounts>`. (The OLD names AF_SANDBOX_SA /
AF_SANDBOX_JOB_DEADLINE_S / AF_SANDBOX_JOB_TTL_S / AF_SANDBOX_LEASE_TTL_S and the omitted WORKSPACE /
LEASE_HMAC_KEY are the bug — codex R-1 #1.) Add a cross-repo test that builds `Settings` from the
rendered env and validates the resulting Job against both VAPs.

## 2. Storage topology — genuinely shared backing across namespaces (codex R-1 #1)
Backing = ONE operator-created NFS export dir `10.55.0.254:/pve-nfs/agentforge-sandbox` (chmod **0777**
so uid 65532 (sandbox) AND the orchestrator/reaper uids all read+write — this also resolves the
NFS-ignores-fsGroup caveat). Static PVs (RWX, `nfsvers=4.0,hard,nconnect=4`, `storageClassName: ""`,
reclaim Retain), one per consumer, ALL pointing at that same server:path:
- **sandbox ns** PV+PVC `af-sbx-ws-<ws>` — sandbox Jobs mount it at `/workspace` `subPath: <job_id>`.
- **tenant ns** PV+PVC `af-sbx-stage-<ws>` — the CP renders this PVC (bound via `volumeName` to the
  operator PV) + mounts it on the orchestrator at `AF_SANDBOX_STAGING_ROOT`. (The PVC is ns-scoped so
  the CP may render it; the PV is cluster-scoped + operator-owned.)
- **reaper ns** (`agentforge`, trusted) PV+PVC `af-sbx-reaper` — the reaper mounts it to read leases +
  rmtree reclaimed dirs.
Per-job isolation stays: the VAP pins `subPath == <job-id label>` and the id is unguessable 32-hex.
The orchestrator works in `<staging_root>/<job_id>/`; the sandbox mounts only that subdir.
**Lease lives OUTSIDE the job-writable subtree**: `<staging_root>/.leases/<job_id>` (NOT mounted into
the sandbox), so a hostile job cannot delete/forge its own lease (codex R-1 #5).

## 3. Reaper — trusted context, one wall-clock, atomic renewal, race-free reclaim (codex R-1 #5)
- Runs in the **`agentforge` namespace** (trusted; NOT under `sandbox-guard`/the zero-egress policy),
  replicas:1, Recreate, mounts the reaper PVC, RBAC = get/list/delete Jobs+Pods in `agentforge-sandbox`
  + a NetworkPolicy allowing the kube-apiserver. Un-gate it once the p1-worker digest is pinned.
- Lease: `time.time()` **everywhere** (agentforge `lease.py` currently stamps `monotonic()` but the
  reaper compares `time()` → every lease reads expired). Renew atomically on a period < TTL while the
  job runs/imports. Reclaim = acquire ownership atomically → recheck Job `.status` AND that no labeled
  Pod exists (foreground delete + watch) → only THEN rmtree the job dir.

## 4. Admission tighten (ailab `sandbox-guard.yaml` + `sandbox-job-guard.yaml`) (codex R-1 #2)
- **Forbid `containers[].envFrom` entirely and `env[].valueFrom` entirely** (no secretKeyRef/
  configMapKeyRef/fieldRef) — else a Job can `secretKeyRef` any Secret in the ns (kubelet resolves it
  without the SA having RBAC) → would leak the reaper lease Secret. R-1 sandbox env is plain values only.
- **Require exact resource requests+limits** (cpu/mem/ephemeral-storage present + within a bounded
  range) and add a namespace **ResourceQuota + LimitRange**.
- Pin the remaining boundary-relevant fields: no `lifecycle`, `stdin`/`tty`, `volumeDevices`,
  `imagePullSecrets`, extra annotations that matter, `schedulerName` override, and require the `home`
  emptyDir `sizeLimit`.
- **Job-guard mirrors the pod-guard**: the Job's `spec.template` must satisfy the SAME container/SC/
  volume/env constraints (not just runtimeClass/SA/restartPolicy).

## 5. Quiescence fail-CLOSED (agentforge `sandbox.py` `_quiesce`) (codex R-1 #3)
`_quiesce()` must **raise on timeout** (never return success), and the caller must NOT import or delete
the job dir on a quiescence failure. Foreground-delete the Job, then **watch** until the Job object is
gone AND no Pod with `agentforge.io/job-id=<id>` exists (treat watch closure / RV-loss as retry-or-fail,
never success). Hold a valid lease until import completes; the `finally` cleanup only runs after proven
quiescence.

## 6. Validator bound to the opened descriptor (agentforge `import_validator.py`) (codex R-1 #4)
`fstat`/`flistxattr` the **opened fd** (not the name), reject any inode/type/nlink/owner/mode change vs
the pre-open lstat, **stream-count actual bytes** (enforce per-file + running total on the copy, not the
stale stat size), **count directories** toward `max_files`/entry bounds, **fail closed** on
`listxattr`/`getxattr` errors, open the result file (`.af-result.json`) with `O_NOFOLLOW` no-follow, and
refuse the path-based degraded walker in production (Linux fd-relative only; the Windows/older path is
dev-test only, gated).
